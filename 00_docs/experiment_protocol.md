# Experiment Protocol

## 1. 场景定义

### 1.1 正常加减速 (Nominal)
- 轨迹：正弦位置跟踪，幅值 1.0 rad，频率 0.5 Hz
- 持续 10 秒
- 无外部接触
- 轻载荷

### 1.2 柔性振动 (Vibration)
- 轨迹：正弦位置跟踪，幅值 0.5 rad，频率 2.0 Hz（接近共振频率）
- 持续 10 秒
- 无外部接触
- 中等载荷

### 1.3 外部冲击 (Collision)
- 轨迹：匀速运动，速度 0.5 rad/s
- 持续 10 秒
- 在 t=5.0s 注入脉冲外力矩
- 轻载荷

### 1.4 故障注入 (Fault Injection)
- 在正常加减速场景基础上叠加：
  - 电流信号卡死
  - 编码器丢包
  - Observer 参数严重失配

## 2. 评价指标

| 指标 | 计算公式 | 目标值 |
|------|----------|--------|
| 位置 RMSE | √(mean((θl_true − θl_est)²)) | < 0.05 rad |
| 速度 RMSE | √(mean((ωl_true − ωl_est)²)) | < 0.3 rad/s |
| 力矩 RMSE | √(mean((τext_true − τext_est)²)) | < 0.3 Nm |
| 检测延迟 | t_detect − t_contact | < 50 ms |
| 误报率 | FP / (FP + TN) | < 5% |
| 漏检率 | FN / (FN + TP) | < 5% |

## 3. 通过标准

- 场景 1.1: 负载侧位置 RMSE < 0.05 rad
- 场景 1.2: 能在 0.3s 内检测到异常并切换模式
- 场景 1.3: 接触检测延迟 < 50ms，减速停止 < 0.5s
- 场景 1.4: 可信评分在故障注入后显著下降（> 0.4 降幅）

## 4. 统计有效性

- 每个随机种子重复 10 次
- 报告均值 ± 标准差
- 随机种子记录在 ValidationReport 中

## 5. 复现要求

- 相同 random_seed 必须产生相同结果
- 仿真环境使用确定性数值积分方法
- 所有配置记录在 configs/ 目录中
