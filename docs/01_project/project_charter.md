# 项目章程

## 项目定义

项目正式名称为：

> **FlexSense-Guard：无负载侧传感器的机器人柔性关节可信虚拟力觉与安全控制仿真系统**

项目只研究一个主导柔性关节。系统在不使用负载侧传感器的条件下，仅依靠电机侧
位置、速度、电流和力矩信号，估计负载侧运动状态与外部关节扰动力矩，评价估计
结果是否可信，并依据正常、柔性振动、外部接触和低可信状态切换控制模式。

当前阶段是可行性验证，不是完整产品开发，也不是硬件验证。

本章程基线日期为 2026-07-21。

## 工业问题

柔性关节中的减速器和弹性传动会造成电机侧与负载侧运动不一致。正常加减速、
柔性振动、摩擦变化和外部接触可能产生相似的电机侧信号；负载变化又会造成模型
失配。增加负载侧传感器会提高硬件成本和维护复杂度，因此项目需要验证仅使用
电机侧信号的虚拟感知方案是否成立，并在估计不可信时进入保守模式。

## 核心目标

1. 建立单柔性关节双惯量数字样机及统一数据契约。
2. 验证仅使用电机侧测量能否重构负载侧位置、速度和外部关节扰动力矩。
3. 验证正常运动、柔性振动和外部接触是否具备可分辨性。
4. 验证估计失效时工程可信评分是否同步下降。
5. 建立事件触发校准、候选参数自动验收及 `UPDATE`、`HOLD`、`ROLLBACK`
   决策框架。
6. 在技术探针通过后，再逐步交付 MATLAB/Simulink、Python、C/C++ SIL、
   展示界面和自动测试报告。

## 作品形态

最终作品规划为纯算法与数字化仿真系统，可包含：

- MATLAB/Simulink 模型与 MIL 验证；
- Python 独立参考实现与契约测试；
- C/C++ 软件在环验证；
- 虚拟传感器展示界面；
- 自动实验、结果管理和报告；
- 测试 Agent。

上述内容是规划交付，不代表当前已经实现。项目当前不依赖真实机器人硬件。

## 必做项

- 单个主导柔性关节的双惯量模型；
- Plant 真值参数与 Observer 名义参数分离；
- 电机侧输入与负载侧真值的严格隔离；
- 负载侧状态和外部关节扰动力矩估计；
- 工程可信评分与可追溯失效原因；
- 统一分类状态和运行模式；
- P1、P2、P3 阶段门及可复现实验证据；
- 失败后的降级路线；
- 跨模块接口、测试和人工审查规则。

## 成功标准

项目不预设算法一定成功。每个探针只能给出 `PASS`、`FAIL` 或
`NOT_VERIFIED`，且必须附带配置、随机种子、提交标识、指标和实际运行记录。

- P1-V 只有在模型、输入和实验设计有效且基于可复验证据通过后，才允许验证 P2。
- P1-A 单独判断 Observer 是否相对直接映射建立稳定优势；P1-A 失败不等于负载侧
  状态不可估计。
- P2 通过后，才允许对外形成接触辨识主张。
- P3 通过后，可信评分才允许驱动安全模式切换。
- 未实际运行的 MATLAB、C/SIL 或其他检查必须标为 `NOT RUN`。
- 任何失败记录不得被覆盖或改写为成功。

## 三级技术路线

### A 路线：完整方案

适用条件：P1-V、P1-A、P2-VIB、P2-CONTACT、P3-DEGRADE、P3-ACTION 和
P3-RECOVERY 均基于有效、充分且可复现的证据通过。

交付目标：

```text
可信虚拟力觉
＋振动/接触辨识
＋事件触发校准
＋安全控制
＋C/C++ SIL
＋展示界面
＋测试 Agent
```

### B 路线：缩减诊断与已验证能力

适用条件：P1-V 通过，但 P1-A、P2 或 P3 至少一项没有通过。

B 路线只保留已经通过相应门禁的能力，不笼统保留全部诊断、可信或控制功能：

- P1-A 未通过时，不声称增广 Observer 优于直接映射；
- P2-VIB 通过而 P2-CONTACT 失败时，只保留振动监测或抑振能力；
- P3 失败时，`confidence_score` 只能用于展示和诊断，不能驱动安全模式；
- 安全动作证据不足时，不形成完整可信安全闭环声明。

P1-V 通过但 P1-A 未通过时进入 `P1_REFRAME_REVIEW`。该状态只能产生三种决定：

```text
CONTINUE_TO_ROUTE_B
ONE_BOUNDED_REPAIR
ENTER_ROUTE_C
```

- `CONTINUE_TO_ROUTE_B`：停止优势主张，只交付已通过门禁的能力；
- `ONE_BOUNDED_REPAIR`：每个 P1 评价活动最多批准一次有限修复，只能处理已定位的
  实现缺陷、输入映射、数值设置，或使用调参集调整预先声明的 Observer 参数；
- `ENTER_ROUTE_C`：主动放弃当前创新主线并转入 SafeTune-J。若 P1-V 已通过，该决定
  是范围和资源选择，不能写成“状态不可估计”。

有限修复不得使用已解封评价集选择 Q/R/P、结构、门限或候选。修复后必须使用新的
独立评价配置和实验 ID；一次修复后 P1-A 仍失败，只能进入 B 或经 Decision ID 批准
进入 C，不得继续循环重定义指标。

### C 路线：SafeTune-J

适用条件：模型、输入和实验设计修复后，P1-V 仍基于有效、可复验的证据无法通过；
或 P1-V 已通过，但团队在 `P1_REFRAME_REVIEW` 中以独立 Decision ID 明确选择放弃
当前主线。后一种情况不得改写 P1-V 的真实结果。

项目降级为“机器人柔性关节事件触发可信自整定与自动验收系统”，保留参数校准、
候选参数生成、自动验收以及更新、保持和回退；取消虚拟负载侧感知、外部扰动力矩
估计和接触识别主张。

## 分阶段规划

| 阶段 | 主要内容 | 主要负责同学 | 进入条件 |
|---|---|---|---|
| 阶段 0 | 章程、架构、接口、门禁、职责、仓库治理和基础 CI | 项目负责人同学 | 当前工作包 |
| 阶段 1 | P1-V/P1-A MATLAB 模型确认、输入修复和多条件复验 | Simulink 同学 | 模型与输入契约共同确认 |
| 阶段 2 | P2 子门和 P3 子门验证 | 通感算同学、项目负责人同学 | P1-V 通过 |
| 阶段 3 | 事件触发校准、自动验收、模式管理和安全控制 | 对应模块同学 | 所需探针通过 |
| 阶段 4 | C/C++ SIL、App、自动报告和测试 Agent | Linux、软件、通感算同学 | 所依赖算法验证通过且接口获批 |
| 阶段 5 | 对比、消融、泛化、故障实验及比赛材料 | 全体同学 | 系统集成完成 |

当前唯一动作以权威状态文件为准。项目负责人先完成规范审查，随后由 Simulink
同学共同确认数学与输入语义，再修改 MATLAB 主实现并复验。项目负责人保留接口、
实验公平性和最终路线决定权，不代写主体实现。

## 声明边界

项目是仿真研究，不包含硬件验证、工业安全认证或未经验证的量化性能声明。当前
核心输出是外部关节扰动力矩，不是末端接触力。公开仓库不得存放专利、比赛敏感
材料或个人数据。
