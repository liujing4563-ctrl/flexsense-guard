# 72 小时技术探针与阶段门

## 目的

72 小时探针用于判断主技术路线是否成立，不是正式功能开发。每个探针必须保存
配置、代码提交、随机种子、指标和原始控制台输出。结论只能是 `PASS`、`FAIL`
或 `NOT_VERIFIED`。

项目负责人同学定义门禁、审查实验公平性并作最终阶段决定；Simulink 同学负责
MATLAB Plant、EKF 和 MIL 的主体实现及实际运行。

## 通用证据要求

- 使用版本化场景配置和固定随机种子；
- 数学模型、力矩所在侧、齿轮方向和符号约定已经共同确认；
- 明确列出 Plant 真实参数与 Observer 名义参数；
- 保存 `torque_command_nm`、`motor_torque_applied_nm` 和
  `motor_torque_measured_nm`，并证明 Plant 与 Observer 输入链路一致；
- 仿真真值只进入评价模块；
- 同一比较使用相同输入数据和评价窗口；
- 报告有限性、发散、误差和运行状态；
- 保留失败结果，不用调低门限换取通过；
- 未实际运行的环境明确写 `NOT RUN`；
- 单次曲线、单一负载或单一随机种子不能作为通过依据。

## P1：负载侧状态估计

### 问题

仅使用电机侧位置、速度和测量力矩，能否在参数失配下稳定估计负载侧位置与速度？

### 负责人

- 项目负责人同学：定义门禁、检查真值泄漏、审查证据和决定路线。
- Simulink 同学：实现并运行 MATLAB Plant、Observer 和 MIL。

### 对比方法

方法 A 为电机侧直接映射：

```text
load_position_baseline = motor_position / gear_ratio
load_velocity_baseline = motor_velocity / gear_ratio
```

方法 B 为既定增广 EKF。P1 阶段不得加入第三种复杂 Observer。

### 实验条件

- 负载：light、medium、heavy；
- 随机种子：至少 0 到 9；
- 工况：名义工况、轻度参数失配、中度参数失配、测量噪声、初始状态失配；
- 失配至少覆盖负载惯量、刚度、阻尼和摩擦；
- Observer 参数不得与 Plant 真实参数完全相同。
- 参数调试集与最终评价集必须按负载、场景和随机种子分离。
- 不得使用最终评价真值选择 Q、R、P 或候选算法。

### 指标

- baseline 与 EKF 的负载位置均方根误差；
- baseline 与 EKF 的负载速度均方根误差；
- 相对改善比例；
- 最大绝对误差；
- 有限性和发散标志；
- 单步或单案例运行时间；
- 每个负载中优于 baseline 的随机种子数量。

### 通过条件

沿用已有门限，不因当前失败降低要求：

先检查证据有效性：

1. 力矩所在侧、齿轮方向、动力学方程和符号已经冻结；
2. 无输入、无外扰、无摩擦和无阻尼条件下通过能量一致性检查；
3. Plant 使用 `motor_torque_applied_nm`，Observer 使用可追溯的
   `motor_torque_measured_nm`，不得直接使用原始指令；
4. Observer 不读取负载侧真值；
5. Observer 名义参数与 Plant 真实参数存在记录明确的失配；
6. 参数调试集与最终评价集分离；
7. light、medium、heavy 和随机种子 0 到 9 均实际运行且可复现。

任一证据有效性条件不满足时，历史运行结果可以记录为 `FAIL`，但当前 P1 可行性
结论必须为 `NOT_VERIFIED`。

只有证据有效后，才评价性能通过条件：

1. 所有实验无 NaN、Inf 或 Observer 发散；
2. 每组 EKF 负载位置均方根误差小于 0.05 rad；
3. light、medium、heavy 的 EKF 位置误差中位数相对 baseline 至少改善 30%；
4. 每个负载至少 8/10 个随机种子优于 baseline；
5. 同时报告负载速度误差、最大误差和运行时间。

证据有效且任一性能强制条件不满足时，P1 为 `FAIL`。只有 MATLAB 主链路与独立
证据均可复验时，才允许把恢复后的 P1 标为 `PASS`。

### 失败后的路线

- 模型、输入或实验设计无效：当前结论保持 `NOT_VERIFIED`，由项目负责人同学
  冻结规则，Simulink 同学修复后重新执行 P1。
- 修复后仍基于有效证据无法稳定通过：启动 C 路线 SafeTune-J。
- P1 未通过期间：P2、P3、App、C/SIL 和测试 Agent 主体开发继续阻断。

## P2：状态可分辨性

### 前置条件

只有 P1 通过并经人工审查后才允许执行。

### 场景

- 正常加减速；
- 柔性振动；
- 摩擦变化；
- 外部冲击。

### 证据

记录外部扰动力矩估计、创新残差、共振频带能量、瞬态能量、分类结果、误报、
漏报和随机种子。正常加减速不得被持续识别为外部接触。

### 失败后的路线

无法可靠区分外部接触和正常动态时启动 B 路线，取消高可信接触识别主张。

## P3：可信失效识别

### 前置条件

只有 P1 通过后才允许执行；涉及接触分类的 P3 场景还要求 P2 通过。

### 故障

- 电流卡死；
- 编码器丢包；
- 时间戳倒退；
- 严重参数失配；
- 执行器饱和。

### 证据

记录真实估计误差、`confidence_score`、`valid_flag`、`reason_codes` 和
`operation_mode`。当估计误差明显增加时，可信评分必须同步下降，并进入
`LOW_CONFIDENCE_DEGRADED`。

### 失败后的路线

可信评分与真实误差不相关时，评分只能作为诊断指标，不得驱动安全模式。
