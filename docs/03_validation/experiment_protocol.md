# 实验协议

## 实验登记

每次实验必须记录：

- 探针编号、场景编号和算法版本；
- Git 提交标识和工作区是否干净；
- 运行环境与工具版本；
- Plant 真实参数和 Observer 名义参数；
- 采样时间、持续时间和初始条件；
- 噪声、故障、参数失配和随机种子；
- baseline、候选方法和共同输入；
- 评价窗口、指标、门限和决定；
- 已运行、失败及未运行的检查。

## 数据隔离

Plant 真值只能进入离线评价。Observer、Confidence、Classification、
Mode Manager 和 Control 不得读取负载侧位置、速度或外扰真值。测试必须检查
输入签名、数据结构和场景运行器没有真值泄漏。

## 公平比较

1. baseline 与候选方法使用同一电机侧输入、同一时间窗口和同一真值。
2. 参数和门限在查看最终评价结果前冻结。
3. 不得只报告最好随机种子、最好负载或最好工况。
4. 不得用评价真值在线选择 Q、R、P 或其他参数。
5. Plant 与 Observer 参数必须分离并保存失配比例。
6. 失败、NaN、Inf 和发散案例必须计入汇总。

## 运行状态

运行状态使用：

- `PASSED`：命令实际运行且通过；
- `FAILED`：命令实际运行但失败；
- `NOT RUN`：环境缺失、阶段门阻断或未执行。

`NOT RUN` 不得写成通过。MATLAB、Octave、Python、C/SIL 和 App 测试分别记录，
一种实现的结果不能替代另一种实现的复验。

## 结论规则

阶段结论使用 `PASS`、`FAIL` 或 `NOT_VERIFIED`。单次图形不能替代重复实验；
无法获得必需证据时使用 `NOT_VERIFIED`，而不是推测。旧失败结果必须保留，并与
当前结果按日期和提交区分。

## 结果保存

- 小型汇总和门禁决定可以纳入版本控制；
- 大量时间序列、构建产物和临时日志不进入源码仓库；
- 机器可读结果必须是合法 JSON，不包含 NaN 或 Inf；
- 报告必须链接配置、结果和对应提交；
- 结果文件不得包含个人数据、专利材料或比赛敏感内容。

## 审查与签字

模块负责同学确认实现、测试和技术说明；验证相关同学确认指标计算和证据完整性；
项目负责人同学检查实验公平性、范围和阶段门，并作最终合并、保持或降级决定。
