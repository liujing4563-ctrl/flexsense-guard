## 决策规则

- `PASS`：证据有效、充分、可复现，并满足运行前批准的全部强制门禁；
- `FAIL`：证据有效，但至少一个强制门禁未满足；
- `NOT_VERIFIED`：模型、输入、实验设计或证据无效、不足、未运行或不可复现；
- `BLOCKED`：前置阶段未通过，当前阶段不得执行主体实验；
- `PROVISIONAL` 只表示工程门限仍需依据确认，不是验证结论。

## P1 证据有效性

1. 模型和符号共同确认；
2. Plant 使用 `PlantInputTrace`，Observer 只读取 `ObserverInput`；
3. `ObserverInput` 不含 applied torque、Plant 真值或真实参数；
4. 无耗散模型通过能量检查；
5. Plant 真实参数与 Observer 名义参数对象分离；
6. light、medium、heavy 与 seed 0 到 9 均实际运行；
7. 调试集和最终评价集分离；
8. 失败、NaN、Inf、发散和提前终止完整保留；
9. 结果关联配置、提交、环境、命令和 artifact index。

任一条件不满足时，P1-V 和 P1-A 只能为 `NOT_VERIFIED`。

## PROVISIONAL 工程门限

| 门禁 | 当前暂存值 | 状态与限制 |
|---|---|---|
| P1-V 位置 RMSE | `< 0.05 rad` | `PROVISIONAL`，不是工业标准，需应用需求确认 |
| P1-V 速度 RMSE | 尚未确定 | 未批准前 P1-V 不得 `PASS` |
| P1-A 位置中位数改善 | `>= 30%` | `PROVISIONAL`，需结合 baseline 水平确认 |
| P1-A 每负载胜出 seed | `>= 8/10` | `PROVISIONAL`，不是统计定理 |
| P2/P3 量化门限 | 尚未确定 | 必须在对应实验前预注册 |

外部扰动力矩 RMSE 可以在 P1 报告中作为诊断指标，但不是 P1-V/P1-A 强制门禁。
