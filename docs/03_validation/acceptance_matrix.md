# 阶段验收矩阵

## 决策规则

- `PASS`：证据有效、充分、可复现，并满足全部冻结门禁；
- `FAIL`：证据有效，但至少一个强制门禁未满足；
- `NOT_VERIFIED`：模型、输入、实验设计或证据无效、不足或不可复现；
- `BLOCKED`：前置阶段未通过，当前阶段不得执行主体工作；
- 单张曲线、单一 seed、Mock 或未运行命令不能决定阶段状态。

## P1 证据有效性前置条件

1. 力矩所在侧、齿轮方向、动力学方程和符号已经冻结；
2. Plant 使用 `motor_torque_applied_nm`，Observer 使用
   `motor_torque_measured_nm`，且逐样本可追溯；
3. 无耗散模型通过能量一致性检查；
4. Observer 不读取负载侧真值，Plant 真实参数与 Observer 名义参数分离；
5. light、medium、heavy 与 seed 0 到 9 均纳入汇总；
6. 调试集和最终评价集分离，不使用评价真值选择 Q/R/P；
7. 历史失败、NaN、Inf、发散和提前终止案例完整保留；
8. 结果关联配置、提交、环境、命令和原始证据。

任一条件不满足时，P1 只能为 `NOT_VERIFIED`，不能据此选择 C 路线。

## P1 现有性能门禁

证据有效后才检查以下条件，且不得因历史失败降低要求：

1. 所有实验无 NaN、Inf 或 Observer 发散；
2. 每组 EKF 负载位置 RMSE 小于 0.05 rad；
3. light、medium、heavy 的位置 RMSE 中位数相对 baseline 至少改善 30%；
4. 每个负载至少 8/10 个随机种子优于 baseline；
5. 同时报告负载速度误差、最大误差和运行时间。
