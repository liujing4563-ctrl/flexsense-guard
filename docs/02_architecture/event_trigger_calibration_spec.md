# 事件触发校准规范

## 状态

规范状态：`DRAFT`。状态、优先级、候选参数边界和回退原则已定义；触发幅值、
持续时间、冷却时间和最终阈值均为 `TBD — requires Plant and P1 evidence`。

## 允许触发条件

- 系统启动或计划维护窗口；
- 工具或负载发生已登记变化；
- 创新残差持续升高；
- 可信评分持续下降但信号仍足以评价；
- 当前参数在独立验收窗口内持续不满足门禁。

## 禁止触发条件

- `valid_flag=false` 或关键测量无效；
- 已处于 `SAFE_SLOWDOWN` 或 `LOW_CONFIDENCE_DEGRADED`；
- 外部接触尚未解除；
- 上一次事件仍在验收、冷却或回退过程中；
- 候选参数越过物理边界，或评价数据不足；
- 使用最终评价集真值在线挑选参数。

## 状态与决定

```text
IDLE -> CANDIDATE -> EVALUATING -> UPDATE
                             \-> HOLD
                             \-> ROLLBACK
```

| 决定 | 条件 | 行为 |
|---|---|---|
| `UPDATE` | 候选在冻结验收窗口内优于当前参数且满足全部安全门禁 | 原子替换并保存前后版本 |
| `HOLD` | 证据不足、无显著改善或仍在冷却 | 保持当前可信参数 |
| `ROLLBACK` | 候选导致失稳、失效、退化或越界 | 恢复最后可信版本并登记原因 |

## 必需保护

- 同一时刻只允许一个候选版本；
- 记录最小事件间隔和冷却窗口，具体值暂为 `TBD`；
- 参数更新必须可追溯、可回滚且不能部分写入；
- 验收失败不得改变当前可信版本；
- 任一异常默认 `HOLD`，已应用候选出现异常则 `ROLLBACK`。

## 证据

每次事件保存触发原因、输入窗口、旧参数、候选参数、冻结门限、评价指标、决定、
提交标识和回退条件。项目负责人定义规则并审查；具体 Plant/EKF 参数候选及实验
由对应模块负责同学产生。
