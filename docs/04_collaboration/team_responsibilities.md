# 同学职责与交接规则

## 总体责任规则

项目负责人同学负责定义架构、接口、验收门禁和集成要求，可以建立最小参考实现
用于独立验证，但不长期代替各模块负责人完成主体实现。

各模块负责人对其模块的源码、测试、实际运行结果、技术说明和交付质量承担主要
责任。Codex 生成代码的责任归属按照代码所在模块确定，不按照发出提示词的人
确定。

每项主体实现必须由对应负责同学解释设计、复现实验并参与 PR 审查。任何同学均
不得以复制代码、代跑命令或只制作图表代替模块责任。

## 项目负责人同学

### 核心职责

- 项目章程、范围、A/B/C 技术路线和最终交付；
- 总体架构、公共接口和跨模块不变量；
- 力矩所在侧、齿轮方向、动力学符号和 Plant/Observer 输入契约；
- P1、P2、P3 门禁、实验公平性和降级决定；
- 事件触发校准、可信门控及 `UPDATE`、`HOLD`、`ROLLBACK` 规则；
- 模式管理、风险控制、系统集成、报告和答辩；
- PR 最终集成审查，不代替模块同学确认主体实现正确。

### 输入接口

- 各模块设计说明、测试结果和风险；
- 公共 Schema 与语言类型；
- 阶段探针配置、指标和运行证据。

### 输出接口

- 冻结的章程、范围、架构、接口和门禁；
- 集成决定、风险处置和路线调整；
- 模块交付入口、审查意见和最终报告。

### 第一阶段交付

- 项目负责人基础工作包；
- 可安装的 Python 公共契约包和基础 CI；
- 五位同学职责、PR 流程和风险登记。

### 验收标准

- 五份核心文档内容一致；
- 公共枚举和字段在文档、Schema 与 Python 中一致；
- P1 真实状态未被篡改；
- 没有把未运行检查写成通过；
- 没有在基础工作包中修改主体算法。

### 前置依赖

- 团队确认项目范围和角色；
- 各模块同学提供接口需求与风险。

### 交接对象

- 将 Plant、Observer 和 MATLAB P1 交给 Simulink 同学；
- 将数据、分类和测试 Agent 交给深度学习通感算同学；
- 将 App、结果管理和自动报告交给计算机软件同学；
- 将 C/C++ SIL 交给嵌入式 Linux 同学。

### 禁止越界

- 不长期主写 MATLAB Plant、EKF、分类器、App、C/SIL 或测试 Agent；
- 不用 Python 参考实现替代模块主实现；
- 不在证据不足时宣布阶段通过；
- 不把个人提示词产生的全部代码记为个人模块成果。

## Simulink 同学

### 核心职责

- 双惯量 Plant、摩擦、饱和、时延、负载变化和外部扰动；
- 与项目负责人同学共同确认力矩侧、齿轮方向、动力学方程和能量一致性要求；
- MATLAB EKF、雅可比、协方差和参数失配；
- Simulink `.slx`、MIL、场景运行、曲线和 MATLAB 证据；
- 当前 P1 MATLAB 修复、复验及结果说明。

### 输入接口

- 冻结的数学模型、三类力矩字段、场景配置和 `MotorSideMeasurement`；
- P1 门禁、负载条件、随机种子和评价协议。

### 输出接口

- `torque_command_nm`、`motor_torque_applied_nm`、
  `motor_torque_measured_nm`、其他电机侧测量、仿真真值和场景元数据；
- `VirtualSensingEstimate` 的 Observer 部分；
- MATLAB/MIL 结果、配置、日志和技术说明。

### 第一阶段交付

- 可复现的三负载双惯量 Plant；
- 只使用电机侧测量的 MATLAB EKF；
- P1 多负载、多随机种子运行记录。

### 验收标准

- 动力学、齿轮比、符号、单位和积分方法说明完整；
- 通过无耗散能量一致性检查；
- Plant 使用实际施加力矩，Observer 使用可追溯测量力矩，不直接使用原始指令；
- Plant 与 Observer 参数分离；
- 无真值泄漏；
- P/Q/R 维度和数值合法；
- MATLAB 实际运行状态与结果如实记录。

### 前置依赖

- 项目负责人冻结模型、接口和 P1 门禁；
- 软件同学提供版本化场景配置格式。

### 交接对象

- 向深度学习通感算同学提供受控场景数据；
- 向嵌入式 Linux 同学提供可复验的 MATLAB 参考；
- 向项目负责人同学提交 P1 证据和风险。

### 禁止越界

- 不修改公共接口而不同步契约；
- 不向 Observer 输入 Plant 真值；
- 不自行降低 P1 门限；
- 不把未运行的 Simulink 或 MATLAB 检查写成通过。

## 深度学习通感算同学

### 核心职责

- 场景数据生成、版本管理和训练/验证隔离；
- 振动与接触特征、规则分类基线和可选轻量三分类模型；
- 故障注入、跨场景泛化、多随机种子数据整理；
- 测试 Agent 主体及证据完整性辅助检查。

### 输入接口

- P1 通过后的状态估计、创新残差和外扰估计；
- 场景配置、仿真标签和数据划分规则；
- P2/P3 门禁。

### 输出接口

- `classification_state` 和 `contact_score`；
- 特征定义、数据版本、评价结果和失效案例；
- 测试 Agent 报告草稿。

### 第一阶段交付

- P2 受控场景数据说明；
- 规则分类基线与无数据泄漏的评价方案；
- 批量实验数据整理脚本设计。

### 验收标准

- 训练、验证和测试数据无泄漏；
- 固定随机种子可复现；
- 正常加速不被持续判为外部接触；
- `contact_score` 明确不是概率；
- P1 未通过时不启动 P2 主体实现。

### 前置依赖

- P1 通过并经审查；
- Simulink 同学提供受控场景和可信运行数据；
- 项目负责人冻结 P2/P3 门禁。

### 交接对象

- 向计算机软件同学提供分类结果和报告字段；
- 向项目负责人同学提交 P2/P3 证据；
- 向嵌入式 Linux 同学说明需移植的特征或规则。

### 禁止越界

- 不让神经网络直接输出控制命令；
- 不使用评价集调门限；
- 不把规则分数称为概率；
- 不负责 App 或 C/SIL 主体实现。

## 计算机软件同学

### 核心职责

- 场景配置、状态管理、结果索引和自动报告；
- 虚拟传感器看板、曲线展示和一键演示；
- 测试管理与用户可见错误处理；
- 早期使用 Mock 数据验证界面契约。

### 输入接口

- 公共 Schema、状态估计、分类、模式和验证报告；
- 版本化场景配置与结果目录约定。

### 输出接口

- 配置文件、运行请求、展示状态和报告；
- App 错误、缺失数据和运行状态提示。

### 第一阶段交付

- 配置与结果管理设计；
- 基于 Mock 数据的界面原型计划；
- 自动报告字段映射。

### 验收标准

- 不修改公共字段语义；
- 正确显示 `NOT RUN`、`FAIL` 和低可信状态；
- 不把占位数据展示为真实实验；
- P1 门禁前不开发完整 App。

### 前置依赖

- 公共接口和报告 Schema 冻结；
- P1 通过后获得真实运行数据。

### 交接对象

- 向项目负责人同学提交集成和演示结果；
- 与深度学习通感算同学对接分类与报告；
- 与嵌入式 Linux 同学对接回放格式。

### 禁止越界

- 不实现 Plant、EKF、分类器或 C/SIL 主体；
- 不用界面效果替代算法证据；
- 不绕过阶段门提前声称完整功能。

## 嵌入式 Linux 同学

### 核心职责

- C/C++ 实现、CMake、CTest 和 SIL；
- MATLAB/Python/C 一致性测试与数据回放；
- 执行时间、内存占用和异常输入保护；
- 边界检查、空指针防护和确定性运行。

### 输入接口

- 冻结的公共契约和可复验 MATLAB 参考；
- 测试向量、容差、采样时间和运行平台约束。

### 输出接口

- C/C++ 库、头文件、构建配置和测试报告；
- 一致性误差、执行时间和内存结果。

### 第一阶段交付

- SIL 接口和移植计划；
- 构建、测试和异常保护验收清单；
- 数据回放格式确认。

### 验收标准

- 编译和 CTest 实际通过；
- 与 MATLAB 参考在声明容差内一致；
- 不使用动态不受控内存；
- 异常输入不会造成崩溃或未定义行为。

### 前置依赖

- P1 通过；
- MATLAB 主实现和公共接口冻结；
- Simulink 同学提供参考数据。

### 交接对象

- 向计算机软件同学提供回放和状态接口；
- 向项目负责人同学提交 SIL 证据和资源报告。

### 禁止越界

- 不在算法未冻结时自行改变模型；
- 不修改公共接口而不同步文档和 Schema；
- 不把仅编译成功写成算法一致性通过。

## PR 与交接规则

1. 每个 PR 只完成一个阶段目标。
2. 主体模块 PR 必须由对应负责同学确认。
3. 公共接口变更必须由所有受影响模块同学审查。
4. 交接必须包含源码、测试、真实运行结果、技术说明和已知风险。
5. 项目负责人同学负责最终集成审查，但不替模块负责同学签署主体实现。
