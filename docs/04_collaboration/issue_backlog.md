## Issue 内容要求

每个 Issue 必须说明：

- 负责同学和交接对象；
- 目标、非目标和允许修改的目录；
- 输入、输出和公共接口；
- 前置阶段门；
- 第一阶段交付与验收标准；
- 实际运行要求和保存证据；
- 风险、降级条件和阻断关系。

历史 P1 执行结果为 `FAIL`，历史证据为 `INVALID / INSUFFICIENT`，当前 P1
可行性结论为 `NOT_VERIFIED`。P2、P3、App、C/SIL 和测试 Agent 主体开发不得
通过新建 Issue 绕过阶段门。
